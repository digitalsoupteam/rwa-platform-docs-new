# Referral & Loyalty Program

The Slices referral program rewards users who bring new participants to the platform. When a referred user pays platform fees, the referrer earns a percentage of those fees.

## How It Works

```
User A (referrer) → refers → User B
                              ↓
User B pays fees (creation, buy, sell)
                              ↓
User A earns referral reward (% of fees)
```

### 1. Register a Referral

When a new user joins, they can register their referrer's ID via the [registerReferral](/docs/api/loyalty/register-referral) mutation. The input takes a single `referrerId` field — the user ID of the referrer.

Rules:
- A user can only register **one referrer** (cannot be changed)
- Users **cannot refer themselves**
- Referral is registered before any fee-generating activity

### 2. Fee Events That Generate Rewards

The referral system tracks four types of on-chain fee events:

| Event | Description | Fee Source |
|-------|-------------|------------|
| **RWA Creation** | Business deploys an RWA token | `createRWAFee` |
| **Pool Creation** | Business deploys a pool | `createPoolFeeRatio` |
| **Buy (mint)** | Participant buys RWA tokens | `entryFeePercent` |
| **Sell (burn)** | Participant sells RWA tokens | `exitFeePercent` |

When any of these events occur on-chain, the indexer picks them up and the loyalty service:

1. Checks if the fee payer has a registered referrer
2. Records the commission in the referrer's fee history
3. Calculates the referral reward as a percentage of the commission

### 3. Referral Reward Calculation

The referral reward is a percentage of the commission amount:

```
rewardAmount = commissionAmount × referralRewardPercentage / 10000
```

The `referralRewardPercentage` is a protocol-level parameter configured in the loyalty service.

### 4. Viewing Fees and Rewards

Users can [query their accumulated fees](/docs/api/loyalty/get-fees) and rewards via GraphQL. The result includes commission amounts and counts for each fee type, plus the total referral reward.

### 5. Withdrawing Rewards

Referral rewards are held in the `ReferralTreasury` smart contract. To withdraw:

1. **Create a withdraw task** — [createReferrerWithdrawTask](/docs/api/loyalty/create-referrer-withdraw-task) requests a multi-signature withdrawal from the treasury
2. **Signers approve** — authorized signers review and sign the withdrawal
3. **Execute on-chain** — once enough signatures are collected, the withdrawal is executed

### 6. Track Withdrawals

View your [withdrawal history](/docs/api/loyalty/get-referrer-withdraws) via GraphQL — each record shows the amount, status, and timestamp.

## Commission History

Every fee event and referral reward is recorded in the [commission history](/docs/api/loyalty/get-referrer-claim-history), providing a full audit trail. Each record includes the action type (`buy_commission`, `sell_commission`, `token_creation_commission`, `pool_creation_commission`, `referral_reward`), amount, transaction hash, and related user wallet.

## Smart Contract

The `ReferralTreasury` contract holds accumulated referral rewards. Withdrawals require multi-signature approval from authorized signers, with an emergency withdrawal function available to Governance.

```solidity
// ReferralTreasury — withdrawal with multi-signature
function withdraw(
    address to,
    uint256 amount,
    uint256 expired,
    address[] calldata signers,
    bytes[] calldata signatures
) external
```
