# Loyalty Program

The Slices loyalty program rewards users who contribute to the platform's growth. It consists of two parts:

- **[Referral Program](/docs/loyalty/referrals)** — earn rewards by referring new users
- **[Airdrop Data](/docs/loyalty/airdrop)** — what on-chain activity is tracked for future distributions

## How It Works

The loyalty service listens to on-chain events and tracks every fee-paying action per user. When a user pays a fee (creation, buy, sell), the system:

1. Checks if the user has a registered referrer
2. Records the commission amount in the user's fee history
3. Calculates a referral reward for the referrer (a percentage of the commission)
4. Stores everything in a queryable database for transparency

## ReferralTreasury Contract

All referral rewards are stored in the **ReferralTreasury** smart contract. This contract holds USDT and is funded **manually** by the protocol — there is no automatic minting or top-up. This means the total reward pool is limited by what has been deposited into the treasury.

Withdrawals from the treasury require **multi-signature approval** from authorized signers, the same system used for RWA and pool deployments.

## Key Data Tracked

The loyalty service tracks the following per user, per chain, per token:

| Metric | Description |
|--------|-------------|
| Buy commissions | Total fees paid when buying RWA tokens |
| Sell commissions | Total fees paid when selling RWA tokens |
| RWA creation commissions | Fees paid when deploying RWA tokens |
| Pool creation commissions | Fees paid when deploying pools |
| Referral rewards | Rewards earned from referred users' fees |
| Referral relationships | Who referred whom |

This data can be used for **future airdrop calculations** — see [Airdrop Data](/docs/loyalty/airdrop) for details.
