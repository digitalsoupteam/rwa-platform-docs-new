# API Endpoints

## Base URL

```
https://api.slices.xyz/graphql
```

## Authentication

All requests require authentication via `Authorization` header:

```bash
curl -X POST https://api.slices.xyz/graphql \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{"query": "{ getBusinesses(input: { filter: {} }) { id name } }"}'
```

Obtain an access token via wallet signature (`authenticate`) or use an API key (`createApiKey`).

## Common Patterns

### FilterInput

Most list queries accept a `FilterInput` with MongoDB-style operators:

```graphql
input FilterInput {
  filter: JSON    # e.g. { ownerId: { $eq: "..." }, riskScore: { $gte: 50 } }
  sort: JSON      # e.g. { createdAt: "desc" }
  limit: Int
  offset: Int
}
```

### Ownership Model

Most entities have `ownerId`, `ownerType`, `creator`, `parentId`, and `grandParentId` fields:

- `ownerType`: `"company"` (businesses/pools owned by companies)
- `parentId`: immediate parent (business ID for pools, business/pool ID for content)
- `grandParentId`: root parent (company ID)
- `creator`: user ID who created the entity

### CreateBusinessInput

```graphql
input CreateBusinessInput {
  name: String!
  ownerId: String!          # Company ID
  ownerType: BusinessOwnerType!  # "company"
  chainId: String!          # "97" for BSC Testnet
  description: String
  tags: [String!]
  country: String           # ISO 3166-1 alpha-2 code
  businessType: BusinessType  # growth | startup | franchise
  socials: [SocialLinkInput!]  # { type, url } — validates URL per type
}
```

### CreatePoolInput

```graphql
input CreatePoolInput {
  name: String!
  businessId: String!
  entryFeePercent: String       # basis points (e.g. "100" = 1%)
  exitFeePercent: String
  expectedHoldAmount: String    # in wei
  expectedRwaAmount: String
  rewardPercent: String         # basis points (e.g. "2000" = 20%)
  entryPeriodStart: Float       # unix timestamp
  entryPeriodExpired: Float
  completionPeriodExpired: Float
  awaitCompletionExpired: Boolean
  floatingOutTranchesTimestamps: Boolean
  fixedSell: Boolean
  allowEntryBurn: Boolean
  priceImpactPercent: String
  outgoingTranches: [OutgoingTrancheInput!]   # { amount, timestamp, executedAmount: "0" }
  incomingTranches: [IncomingTrancheInput!]   # { amount, expiredAt, returnedAmount: "0" }
  description: String
  tags: [String!]
}
```

### Approval Signatures Flow

```graphql
# 1. Request signatures
mutation {
  requestBusinessApprovalSignatures(input: {
    id: "business_id"
    ownerWallet: "0x..."
    deployerWallet: "0x..."
    createRWAFee: "100"
  }) { taskId }
}

# 2. Poll for completion (signers sign off-chain)
query {
  getSignatureTask(input: { taskId: "task_id" }) {
    completed
    signatures { signer signature }
    expired
  }
}

# 3. Call Factory.deployRWA() on-chain with collected signatures
```
